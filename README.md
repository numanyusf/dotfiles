# Dotfiles

Personal configuration files for my **Ubuntu (GNOME)** setup ‚Äî Lenovo Legion S7.

Theme across everything: **One Dark** (coral `#E06C75`, gold `#E5C07B`, green `#98C379`) with the GNOME/Yaru **orange** system accent.

## Reinstall in one command

On a fresh Ubuntu install:

```bash
git clone https://github.com/numanyusf/dotfiles.git ~/.dotfiles
cd ~/.dotfiles && git checkout ubuntu
./bootstrap.sh
```

`bootstrap.sh` adds the third-party apt repos (1Password, VS Code, Firefox, eduVPN), installs everything in `packages.txt`, installs oh-my-posh + the Nerd Font, lays all the symlinks, sets up tmux/TPM, and restores Ptyxis settings. It is idempotent ‚Äî safe to re-run.

It does **not** do the system-level security setup (LUKS/TPM2 unlock, YubiKey tap-to-login) ‚Äî those touch `/etc`, need physical touches or secrets, and are documented step-by-step in **[`docs/system-setup.md`](docs/system-setup.md)**. Do those after bootstrap; the script prints a reminder.

## Included configs

- `bashrc` ‚Äî Bash config: inits **oh-my-posh**, loads `ls_colors`, and aliases `ls`‚Üí**eza** (with Nerd Font icons)
- `gitconfig` ‚Äî git identity + **1Password SSH commit signing** + gh credential helpers ‚Üí `~/.gitconfig`
- `ssh/config`, `ssh/allowed_signers` ‚Äî SSH hosts (CSC lumi/puhti/roihu, etc.) + signing pubkeys ‚Üí `~/.ssh/`
- `1password/agent.toml` ‚Äî 1Password SSH-agent vault exposure ‚Üí `~/.config/1Password/ssh/agent.toml`
- `oh-my-posh/` ‚Äî prompt theme `emodipt-extend.omp.json` (time, shell, git, node/language versions, project version, battery, python venv, SSH host, path, exec-time, status)
- `ls_colors` ‚Äî `LS_COLORS` for `ls`/eza, generated by **vivid** (`one-dark` theme) with the directory color overridden to gold `#E5C07B`; `bashrc` loads it directly (no symlink needed)
- `ptyxis.dconf` ‚Äî Ptyxis terminal settings (font, opacity, palette, copy toast)
- `nvim/` ‚Äî Neovim (LazyVim) with the **One Dark** colorscheme (`navarasu/onedark.nvim`)
- `tmux/` ‚Äî tmux config with TPM plugins (prefix `C-a`)
- `packages.txt` ‚Äî apt package manifest installed by `bootstrap.sh`
- `scripts/remove-luks-pin.sh` ‚Äî re-bind TPM2 to LUKS after a firmware/PCR-7 change (see the doc)
- `docs/system-setup.md` ‚Äî LUKS+TPM2 unlock, YubiKey PAM, hardware quirks, backup checklist
- `docs/oh-my-posh-portable.md` ‚Äî standalone handout: replicate just the prompt on any distro (Arch/Fedora/‚Ä¶), terminal, and shell

## Manual setup (what `bootstrap.sh` automates)

Prefer `./bootstrap.sh`. The equivalent by hand:

```bash
ln -sfn ~/.dotfiles/bashrc              ~/.bashrc
ln -sfn ~/.dotfiles/gitconfig           ~/.gitconfig
ln -sfn ~/.dotfiles/oh-my-posh          ~/.config/oh-my-posh
ln -sfn ~/.dotfiles/nvim                ~/.config/nvim
ln -sfn ~/.dotfiles/tmux                ~/.config/tmux
ln -sfn ~/.dotfiles/ssh/config          ~/.ssh/config
ln -sfn ~/.dotfiles/ssh/allowed_signers ~/.ssh/allowed_signers
ln -sfn ~/.dotfiles/1password/agent.toml ~/.config/1Password/ssh/agent.toml
```

(`ls_colors` needs no symlink ‚Äî `bashrc` reads `~/.dotfiles/ls_colors` directly. Regenerate with `vivid generate one-dark | sed 's/di=[^:]*/di=0;38;2;229;192;123/' > ~/.dotfiles/ls_colors`.)

Restore Ptyxis settings:

```bash
dconf load /org/gnome/Ptyxis/ < ~/.dotfiles/ptyxis.dconf
```

tmux plugins:

```bash
git clone https://github.com/tmux-plugins/tpm ~/.tmux/plugins/tpm
~/.tmux/plugins/tpm/bin/install_plugins
```

nvim plugins install automatically on first launch (LazyVim bootstraps `lazy.nvim`).

## Notes

- The terminal directory color, oh-my-posh path color, and `ls`/eza directory color are all kept in sync at gold `#E5C07B` ‚Äî every other file type gets a vivid one-dark color, so only directories use the gold.
- To change the nvim One Dark variant, edit `style` in `nvim/lua/plugins/colorscheme.lua` (`dark`, `darker`, `cool`, `deep`, `warm`, `warmer`).
