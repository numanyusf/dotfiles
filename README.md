# Dotfiles

Personal configuration files for my **Ubuntu (GNOME)** setup — Lenovo Legion S7.

Theme across everything: **One Dark** (coral `#E06C75`, gold `#E5C07B`, green `#98C379`) with the GNOME/Yaru **orange** system accent.

## Reinstall in one command

On a fresh Ubuntu install:

```bash
git clone https://github.com/numanyusf/dotfiles.git ~/.dotfiles
cd ~/.dotfiles && git checkout ubuntu
./bootstrap.sh
```

`bootstrap.sh` adds the third-party apt repos (1Password, VS Code, Firefox, eduVPN), installs everything in `packages.txt`, installs oh-my-posh + the Nerd Font, lays all the symlinks, sets up tmux/TPM, and restores Ptyxis settings. It is idempotent — safe to re-run.

It does **not** do the system-level security setup (LUKS/TPM2 unlock, YubiKey tap-to-login) — those touch `/etc`, need physical touches or secrets, and are documented step-by-step in **[`docs/system-setup.md`](docs/system-setup.md)**. Do those after bootstrap; the script prints a reminder.

**For the full end-to-end walkthrough** — fresh install choices, firmware, bootstrap, 1Password, git signing, LUKS/TPM, YubiKey, and a final verification checklist, all in order — see **[`docs/ubuntu-setup-guide.md`](docs/ubuntu-setup-guide.md)**.

## Included configs

- `bashrc` — Bash config: inits **oh-my-posh**, loads `ls_colors`, and aliases `ls`→**eza** (with Nerd Font icons)
- `gitconfig` — git identity + **1Password SSH commit signing** + gh credential helpers → `~/.gitconfig`
- `ssh/config`, `ssh/allowed_signers` — SSH hosts (CSC lumi/puhti/roihu, etc.) + signing pubkeys → `~/.ssh/`
- `1password/agent.toml` — 1Password SSH-agent vault exposure → `~/.config/1Password/ssh/agent.toml`
- `oh-my-posh/` — prompt theme `emodipt-extend.omp.json` (time, shell, git, node/language versions, project version, battery, python venv, SSH host, path, exec-time, status)
- `ls_colors` — `LS_COLORS` for `ls`/eza, generated by **vivid** (`one-dark` theme) with the directory color overridden to gold `#E5C07B`; `bashrc` loads it directly (no symlink needed)
- `ptyxis.dconf` — Ptyxis terminal settings (font, opacity, palette, copy toast)
- `nvim/` — Neovim (LazyVim) with the **One Dark** colorscheme (`navarasu/onedark.nvim`)
- `tmux/` — tmux config with TPM plugins (prefix `C-a`)
- `packages.txt` — apt package manifest installed by `bootstrap.sh`
- `scripts/remove-luks-pin.sh` — re-bind TPM2 to LUKS after a firmware/PCR-7 change (see the doc)
- `docs/ubuntu-setup-guide.md` — **full end-to-end runbook** for this machine (install → verified), stitching bootstrap + the docs below in order
- `docs/system-setup.md` — LUKS+TPM2 unlock, YubiKey PAM, hardware quirks, backup checklist
- `docs/oh-my-posh-portable.md` — standalone handout: replicate just the prompt on any distro (Arch/Fedora/…), terminal, and shell

## Dependencies

- **Ptyxis** terminal (GNOME)
- **oh-my-posh** — `curl -s https://ohmyposh.dev/install.sh | bash -s` (installs to `~/.local/bin`)
- **eza** — `sudo apt install eza`
- **vivid** — `sudo apt install vivid` (only needed to *regenerate* `ls_colors`; the committed file works standalone)
- **MesloLGM Nerd Font** — installed under `~/.local/share/fonts/Meslo/`
- **Neovim** ≥ 0.11

## Manual setup (what `bootstrap.sh` automates)

Prefer `./bootstrap.sh`. The equivalent by hand:

```bash
ln -sfn ~/.dotfiles/bashrc              ~/.bashrc
ln -sfn ~/.dotfiles/gitconfig           ~/.gitconfig
ln -sfn ~/.dotfiles/oh-my-posh          ~/.config/oh-my-posh
ln -sfn ~/.dotfiles/nvim                ~/.config/nvim
ln -sfn ~/.dotfiles/tmux                ~/.config/tmux
ln -sfn ~/.dotfiles/ssh/config          ~/.ssh/config
ln -sfn ~/.dotfiles/ssh/allowed_signers ~/.ssh/allowed_signers
ln -sfn ~/.dotfiles/1password/agent.toml ~/.config/1Password/ssh/agent.toml
```

(`ls_colors` needs no symlink — `bashrc` reads `~/.dotfiles/ls_colors` directly. Regenerate with `vivid generate one-dark | sed 's/di=[^:]*/di=0;38;2;229;192;123/' > ~/.dotfiles/ls_colors`.)

Restore Ptyxis settings:

```bash
dconf load /org/gnome/Ptyxis/ < ~/.dotfiles/ptyxis.dconf
```

tmux plugins:

```bash
git clone https://github.com/tmux-plugins/tpm ~/.tmux/plugins/tpm
~/.tmux/plugins/tpm/bin/install_plugins
```

nvim plugins install automatically on first launch (LazyVim bootstraps `lazy.nvim`).

## Notes

- The terminal directory color, oh-my-posh path color, and `ls`/eza directory color are all kept in sync at gold `#E5C07B` — every other file type gets a vivid one-dark color, so only directories use the gold.
- To change the nvim One Dark variant, edit `style` in `nvim/lua/plugins/colorscheme.lua` (`dark`, `darker`, `cool`, `deep`, `warm`, `warmer`).
